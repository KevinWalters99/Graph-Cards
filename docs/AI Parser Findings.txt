Push In The Right Direction (Gemini)
------------------------------------------------------------

Recommended implementation order:

1) Add AI parser config storage (DB migration)
- Create `CG_AiParseConfig` with:
  - provider (`programmatic|gemini|grok`)
  - gemini_api_key (encrypted/at least masked in UI)
  - grok_api_key (same)
  - model_name
  - temperature
  - max_output_tokens
  - enabled flag
  - updated_by/updated_at

2) Add provider abstraction in backend
- Introduce `parseWithProvider($provider, $fullText, $refData)` called from `parseSession`.
- Keep current deterministic parser as fallback/default.

3) Implement Gemini adapter first
- Use server-side HTTP call with:
  - strict JSON schema output contract
  - timeout + retry with exponential backoff for 429/5xx
  - parse/validate JSON before DB insert
- If Gemini fails, fail run with actionable error or fallback to deterministic mode (decide explicitly).

4) Add minimal admin UI controls
- Provider selector + per-provider masked key fields
- "Test Connection" action for Gemini/Grok
- Model dropdown/text input

5) Add logging and observability
- Store provider, model, latency, token counts (if returned), HTTP status, retry count.
- Persist error body snippets safely for debugging (truncate/redact keys).

6) Add integration tests for parser contract
- Given transcript input, assert:
  - JSON contract validity
  - insert succeeds
  - parse run status transitions correctly
  - fallback behavior (if enabled)

------------------------------------------------------------
Immediate Practical Note
------------------------------------------------------------

Given this repo state, Gemini failures are currently not from a bad Gemini implementation; they are from absence of an integrated Gemini pipeline in the active parse path.
